Java SE 总览

虽然以后要从事的是 Java EE 的开发工作，但作为 ME 和 EE 的基础，SE 的一些重点以及细节的掌握也很重要，这篇文章不会有太多的代码和细节部分，因为文章的目的是总结而不是庖丁解牛般的深究。

JDK 与 JRE

  • JDK(Java Development Kit) : Java 开发工具包，用来开发 Java 程序，它包括了 JRE 和开发所要用到的工具类和工具类 API

  • JRE(Java Runtime Environment) : Java 运行环境，用来运行 Java 程序，如果只是单纯的想要运行 Java 程序，是不需要 JDK 的。JRE 包含 JVM 与核心类库。

数据类型

Java 是一种静态类型的语言，这意味着我们在使用所有的变量时都必须要对它进行声明和初始化。按照类型分，Java 的数据类型可以分为基本数据类型以及引用数据类型

  • 基本数据类型 ：不需要通过 new 关键字来创建的数据类型，Java 内置八种基本数据类型，如下。

    | 类型 | 字节长度 | 默认值 | 取值范围 |
    | :—— | :— | ——– | —————————– |
    | BYTE | 1 | 0 | -128 ~ 127 |
    | SHORT | 2 | “” | -32768 ~ 32767 |
    | CHAR | 2 | “\u0000” | “\u0000”(0) ~ “\uffff”(65535) |
    | INT | 4 | 0 | $-2^{31}$ ~ $2^{31}-1$ |
    | LONG | 8 | 0L | $-2^{63} $ ~ $ 2^{63}-1$ |
    | FLOAT | 4 | 0.0f | $-2^{31}$ ~ $2^{31}-1$ |
    | DOUBLE | 8 | 0.0d | $-2^{63} $ ~ $ 2^{63}-1$ |
    | BOOLEAN | 1 | FALSE | true / false |

  • 引用数据类型 ： 不能够直接赋值，必须通过变量地址来引用的数据类型，如类、接口、数组都属于引用数据。

堆与栈

当 JVM 在执行 Java 程序时，它会把自己所管理的内存部分划分为 5 个单元，我们统称它们为数据区。

其中，方法区和堆为共享数据区，即在多线程程序中，它们的方法区与堆里面的数据时共享的，而栈空间则为私有的，JVM 在栈空间中执行线程中的方法。

  • 方法区 Method Area（共享）：用来保存加载类的类信息，包括类名，类的常量池，还有类中的方法及变量等，当类中含有静态方法或变量时，为静态方法或变量分配内存空间，外界可以直接访问。而不需要 new 出对象来。此外，在加载类时还会在区中生成该类对应的一个 Class 对象。
  • 堆 Heap（共享）：堆是 5 个数据单元区中最大的一个单元，它随着 JVM 的启动而创建，所有线程共享，它用于保存类和数组等通过 new 关键词创建出来的实例对象，同时具备内存空间的地址用于栈来访问。在栈中创建的实例对象中的变量或类都具备初始化变量值，引用数据类型默认为 null ，基本数据类型默认为默认类型。
  • 栈 Stack （私有）：栈是线程所独有的，每创建一个线程就会随之创建一个对应的栈，它随着线程的启动而创建，线程的销毁而销毁。在栈中创建的对象或变量不具备初始化值，必须手动初始化，否则无法使用。栈的主要作用是用来执行方法，且只能执行位于栈顶的方法，当运行中的方法调用了其他方法时，该方法即进行压栈，调用方法入栈。由于栈的特点，它在运行时可能会出现两个异常： stackoverflow 与 OutOfMemoryError。
  • 本地方法栈 Native Method Stack（私有）：也是一种栈，主要用于运行本地系统语言编写的修饰符为 naive的方法。
  • 程序计数器 Program Counter Register （私有） ：是 5 个数据单元中内存空间最小的单元，它的作用是指示虚拟机的执行顺序，它的内部根据字节码文件依次保存当前所执行指令的下一条字节码指令的地址。

数组排序

虽然在实际开发中很少用到数组，基本上都是用的集合，但是在某些场合下，数组还是有其不可替代性的，对数组最常见的操作便是排序了，常用的排序算法有选择排序、冒泡排序、 折半查找、快速排序。

  • 选择排序 ：每次比较保存最小值的下标索引，等一次内循环完成后再进行索引值的交换

  • 冒泡排序 ：冒泡排序与选择排序的思想是一样的，不过它的实现方式是每次比较后即彼此交换索引值，因此效率比选择排序要低些。

  • 选择排序 ：选择排序的排序思想是通过不停递归切分来进行排序，保证每次递归排序后产生的切分点的左侧都小于原切分点的索引值，切分点的右侧都大于原切分点的索引值。这种通过不停的切分来进行排序的算法效率远远高于普通的选择排序，是前者的 5~6 倍

关键字

  • this : 表示对自身所属对象的引用。每个函数内部都有一个隐藏的 this ，它指示的是所属函数的地址值，因此可以通过这个 this 来访问类的成员变量。
    1. 作用一：区分成员变量与局部变量。
    2. 作用二：调用本类构造函数（必须放在第一行）。
    3. 作用三：返回自身的引用，构成引用链（构造者模式）。
  • static : 被 static 修饰的函数或成员变量保存在方法区中，它随着类的加载而在方法区中开辟空间，因此不需要创建对象即可以通过类名直接进行调用。由于这种特点，因此 static 内部不能包含与对象创建后才拥有的属性，例如 this ，成员变量，成员函数等。
  • final : 最终的，最后的。被修饰的类不可继承，变量不可改变（相当于常量），修饰的函数不可以在子类中进行复写，但可以在子类中直接调用。
  • abstract : 抽象，被 abstract 修饰的类与函数需要子类继承与重写后才有实际作用，由于这种特性，因此该关键字不能与 private 与 final 共存。

封装继承多态

  • 封装 : 隐藏类中成员属性的细节，外界不可以直接访问，必须通过 set() 与 get() 方法才能进行访问，它能使类变得更安全与易于管理。
  • 继承 : 一个类继承另一个类，如果父类是抽象类，那么必须实现父类中的所有抽象方法，继承用于描述一类事物。
  • 多态 : 成员变量与静态成员函数看父类，只有成员函数先看子类，子类没有的话就运行父类中的，如果父类也没有，那么就会在编译时就抛出异常。

内部类

内部类是表示一个类中的类，它用于描述一个类的附属信息，可以定义在成员位置或成员函数内部，这个类外界不能直接直接访问，必须用 new Outer().new Inner() 来访问。不过，当内部类的修饰符为 static 时，由于在加载时就已经保存在方法区了，因此可以通过 new Outer.Inner() 来创建对象。

异常

异常用于描述在程序编译或执行过程中可能会发生的错误信息。

  • 异常的分类

  • 自定义异常：自定义异常内部不需要任何实现，只需要实现构造方法即可。

  • throw 与 throws ：

    • throw : 抛出异常，函数体中。抛出的是编译时异常后必须在函数名进行声明（即 throws ） 或直接捕获。
    • throws : 声明异常，函数名上。调用或者继承有异常声明的函数或类必须捕获或继续声明该异常。

类型转换

集合

集合可以说是 Java 语言中最令人兴奋的特性之一了，同样是存储数据，它的作用比一般的数组要强大的多，长度可变，各种富有特性的函数更是让对数据的操作更为便利与简单。

  • 集合继承体系图

    • collction : 属于单列集合的顶级接口，实现类都可以进行 iterator 迭代,线程不安全（Vector 安全）。

      ​ |—- List : 存取有序。有索引，可包含重复元素。

      ​ |—- ArrayList : 底层数据结构是数组，增删慢，查询快。

      ​ |—- LinkedList : 底层数据结构是链表，增删快，查询慢。

      ​ |—- Vector （几乎不用）: 底层结构是数组，增删查询都慢。

      ​ |—- set : 存取无序（LinkedHashSet 有序），不包含重复元素，可以存 null 元素，无索引。

      ​ |—- HashSet : 底层数据结构是哈希表（实际上是 HashMap 实例），如果是引用对象则需要复写 hashCode() 与 equals() 。

      ​ |—- TreeSet : 底层数据结构是红黑树，存储的时候按照自然排序或者传入的 comparator 进行比较排序 。

      ​ |—- LinkedHashSet : 可预知迭代顺序，链表保存插入顺序，Hash 表保存数据。它通常用于保存一份已知 Set 集合的副本，使这个副本的迭代顺序与插入顺序相同。

  • Map : 属于双列集合的顶级接口，键值对存储。不可以直接迭代，必须先生成 Collection 集合然后再遍历。线程不安全（HashTable 安全）。

    ​ |—- HashMap : 基于 Hash 表，主要作用在键，保证键唯一 。

    ​ |—- TreeMap : 基于红黑树，按自然順序或者傳入的 comparator 來保证 Map 中元素的迭代順序。

    ​ |—- LinkedHashMap : 特性同 LinkedHashSet 。

    ​ |—- HashTable （几乎不用） : 线程安全，不允许 null 元素。

IO

  • File : 文件和目录路径名的抽象表现形式，相当于将指定路径的文件或文件夹封装成一个类，方便对其进行操作。

    ​ |—- delete() : 删除文件或文件夹，不走回收站，当且仅当文件夹为空时才能进行删除。

    ​ |—- createNewFile() : 将封装的文件路径在硬盘上创建对应的文件，指定文件存在返回 false 。

    ​ |—- listFiles() : 返回调用对象内的所有文件与文件夹，当对象为文件或发生 I/O 错误返回 null 。

    ​ |—- isFile() : 判断调用对象时否为文件, isDirectory() 判断调用是否为文件夹。

    ​ |—- FileFilter : 文件过滤器， accept(File pathName) 函数内来过滤该指定文件名的文件。

  • 流

    ​ |—- InputStream: 以字节为单位来进行读数据的抽象类。

    ​ |—- FileInputStream :文件字节输入流

    ​ |—- ObjectInputStream : 序列化对象输入流，本身不具备读能力

    ​ |—- ObjectInputStream(InputStream in) : 接受一个 InputStream 对象，通过该对象从持久化设备中的读取数据。

    ​ |—- FilterInputStream : 本身不具备读能力，通过对传入的 InputStream 进行一些装饰（装饰者模式），提高

    InputStream 的读能力

    ​ |—- BufferedInputStream : 缓冲输入流，内部有一个缓冲数组，用于暂存读入的数据。

    ​ |—- OutputStream: 以字节为单位来进行读数据。

    ​ |—- FileOutputStream :文件字节输出流

    ​ |—- ObjectOutputStream : 序列化对象输出流，本身不具备写能力

    ​ |—- ObjectOutputStream(OutputStream out) : 接受一个 OutputStream 对象，并通过该对象向持久化设备写入数据。

    ​ |—- FilterOutputStream : 本身不具备读能力，通过对传入的 OutputStream 进行一些装饰（装饰者模式），提高 OutputStream 的读能力

    ​ |—- BufferedOutputStream : 缓冲输出流，内部有一个缓冲数组，用于暂存写入的数据。

    ​ |—- PrintStream : 字节打印流，永远不会抛出 IOException

    ​ |—- Reader : 以字符为单位来进行读数据的抽象类

    ​ |—- InputStreamReader : 字符流与字节流之间的输入转换流

    ​ |—- FileReader : 文件字符输入流

    ​ |—- Writer : 以字符为单位来进行写数据的抽象类

    ​ |—- OutputStreamWriter: 字符流与字节流之间的输出转换流

    ​ |—- FileWriter : 文件字符输出流

    ​ |—- PrintWriter : 打印流，向网络、文件、控制台打印数据，只能打印 char 类型数组，其他类型的数组会直接打印其地址值，需要开启自动刷新，且需要调用 println() printf() format() 三个函数之间一个才会刷新。该方法永远不会抛出 IOException 。

线程

线程是并发的基础，线程可以让程序在同一时间段“并发”的运行两个任务。JVM 启动后至少运行了两条线程，一条主线程，一条垃圾回收线程。

线程的创建

  1. 继承 Thread 类，复写 run() 方法。

  2. 实现 Runable 接口，复写 run() 方法。

线程的控制

​ |—- setPriority() : 设置线程的优先级（概率事件）

​ |—- sleep(long time) : 线程休眠指定时长

​ |—- join() : 加入线程，相当于插队，该线程执行完其他线程才可以继续运行

​ |—- yield() : 线程礼让，与 sleep 类似，但时间随机

同步锁

  • synchronize : 代码同步块，该关键字通过某个对象锁可以锁住一段代码块或者一个方法，这个锁对象必须唯一，当要锁住是代码块时，使用 new Object() 充当锁对象，是非静态方法时，锁对象默认为 this 。而在静态方法时，锁对象可以指定为类名.class。

  • lock : 锁接口，可以直接锁住一段代码块而不需要锁对象，因为它本身就相当于一把锁。它支持多个支持休眠与唤醒线程的 condition 对象。

    ​ |—- newCondition() : 获取一个 condition 对象，该对象可以休眠与唤醒线程，但能且只能唤醒是自己所休眠的线程，而不能唤醒其他 condition 对象休眠的线程。

线程池

  • ExecutorService: 线程池实现接口

    ​ |—- Executors.newCachedThreadPool() : 通过 Executors 工具类来获得一个可缓存的线程池对象，该对象是一个 ExecutorService 的实现类，获得的线程池对象可以保存使用过的线程，当线程池中没有线程时就创建线程，当有时就直接返回已缓存的线程对象。

    ​ |—- Executors.newFixedThreadPool() : 通过 Executors 工具类来获得一个固定线程数的线程池对象，该对象是一个 ExecutorService 的实现类。

网络编程

概述

利用程序进行网络数据的传送与接收，最重要的三个元素：IP、端口、协议。

InetAddress

​ |—- getAllByName() : 获取指定主机名下的所有 IP 地址

​ |—- getByName() : 在给定主机名的情况下获取主机的 IP 地址

Socket

网络通信实际上就是 Socket 之间的通信，通信两端都具有一个 Socket ，数据通过 I/O 技术在两个 Socket 之间进行传输。可以通过 Socket 封装目标 IP 与端口进行数据的传输。

​ |—- Socket(InetAddress address, int port) : 将创建的 Socket 对象绑定到指定的目标 IP 与端口。

UDP

UDP 是无连接的一种通信方式，发送数据前不需要先进行连接，类似于现实生活中的发送快递，先创建一个数据包，然后获得一个发送对象，发送该数据包。

  • DatagramPacket

    ​ |—- DatagramPacket(byte[] buf, int offset, int length, InetAddress address, int port) : 发送数据报包，包含数据发送的目标 IP 与端口，该报包可以携带一个指定偏移量字节数组的数据，但偏移数组容量最大不能超过 64kB。

    ​ |—- DatagramPacket(byte[] buf, int length) : 接收数据报包，接收一个接收数组与数组长度，将接收的数据保存到接收到的字节数组中。

  • DatagramSocket

    ​ |—- DatagramSocket() : 创建一个随机端口的 DatagramSocket 对象。

    ​ |—- DatagramSocket(int port) : 创建一个监听指定端口的 DatagramSocket 对象。

    ​ |—- send(DatagramPacket p) : 接收一个数据报包，发送到报包指定的目标主机与端口。

    ​ |—- receive(DatagramPacket p) : 将从 DatagramSocket 接收对象中接收的数据报包对象保存到指定的数据报包中。

TCP

TCP 是面向连接的通信方式，它在进行数据传输前必须先进行通信双方的连接，因此需要进行三次握手，所以效率较低，不过安全可靠，且发送数据没有大小限制。

  • Socket

    ​ |—- Socket(InetAddress address, int port) : 将创建的 Socket 对象绑定到指定的目标 IP 与端口。

    ​ |—- getInputStream() : 获取输入流，读取数据

    ​ |—- getOutputStream() : 获取输出流，输出数据

    ​ |—- shutDownOutput() : 结束输出流的输出，用于给服务度一个结束标志，终止服务端的阻塞状态

  • ServerSocket

    ​ |—- ServerSocket(int port) : 创建一个监听指定主机端口的 ServerSocket 对象。

    ​ |—- accept() : 返回一个可读写数据的 Socket 对象。

URL

  • URL网络的统一资源定位符，即网址，该对象可以封装一个网址，并进行连接。

    ​ |—- URL(String spec) : 创建指定域名的 URL 对象

    ​ |—- openConnection() : 打开连接，返回一个 connection 对象，这个对象代表着应用程序与对应 URL 之间的通信连接，可以通过它来获取双方通信用的输入输出流。

    ​ |—- openStream() : 打开连接，打开输入流，返回一个 InputStream 对象

反射

反射是一种用于在运行时操作类信息的技术，通过反射可以在运行时动态的改变某个类的状态。反射的是基于字节码对象实现的，当某个类被加载时，在内存的方法区中就会有这个某的字节码对象，即 Class 对象，这个对象包含这个类的所有类型信息，包括所有的字段、构造函数、成员方法等。

概述

  • 获取 Class 对象的三种方法

    ​ |—- Class.forName("...") : 通过某个类的完整类名来获取

    ​ |—- obj.getClass() : 通过对象继承之 Object 的 getClass() 方法来获取

    ​ |—- Obj.class : 通过类名.class 来获取

  • 反射的主要方法

    ​ |—- getConstructor(Clas<?>... parametertype) : 通过具体的参数字节码对象信息获取对应的构造函数

    ​ |—- getMethod(String name, Class<?>... parametertype) : 通过方法名以及具体的参数字节码对象信息获取对应的方法

    ​ |—- getField(String name) : 通过字段名来获取对应的字段

    ​ |—- getClassLoader() : 获取类的加载器对象

Proxy

代理模式是反射的一个很好的应用，通过反射技术，可以实现适应多场景且操作十分方便的动态代理。要注意的是，Proxy 是基于接口的动态代理，也就是或某个类要实现多态代理，那么他们必须是实现同一接口而非同一抽象父类。

​ |—- newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h) : 通过指定的 handler 来生成某个对象的代理对象，返回该对象的代理对象。

​ |—- invoke(Object proxy, Method method, Object[] args) : 实现自定义 handler 时要完成的方法，该方法可以在指定方法运行前做一些附加操作，也可以称其为实际方法的代理方法。
